## Why
Guide users through adding docstrings for missing or weak documentation findings.

## What Changes
- Run doc analyzer and filter for missing_docstring (and weak_docstring when --mode all).
- Let users pick modules or all in scope, generate docstring plans, and optionally apply via existing flow.
- Emphasize short, factual docstrings without behavior changes.

## Impact
- Streamlines documentation cleanups with optional automation.
- Keeps docstring changes factual and scoped.
